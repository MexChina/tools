go run  原理：编译 到临时  运行编译结果  可执行文件，归档文件

-a  强制编译相关代码，不论他们的编译结果是否最新的
-n 打印编译过程中所需运行的命令，但不真正执行他们
-p n 并行编译，其中n为并行的数量，建议n为逻辑cpu的个数  4核
-v 列出被编译的代码包的名称
-work 显示编译时创建的临时工作目录的路径，并且不删除它
-x 打印编译过程中所需运行的命令


go build 编译源码文件或代码包
		 生成可执行文件

go install 编译并安装代码包或源码文件
